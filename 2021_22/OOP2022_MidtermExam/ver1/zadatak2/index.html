2. zadatak (7 bodova)

Vaš zadatak je napisati programski kod na način kako je to specificirano UML dijagramom klasa na slici.

Legenda vidljivosti u dijagramu klasa:

Napomena: Sve klase i sučelja imaju vidljivost postavljenu na package private i nalaze se u paketu hr.fer.oop.mtexam.task2.

Klasom MultiAgentSystem je oblikovan višeagentski sustav koji može sadržavati 0 ili više programskih agenata. Maksimalan broj agenata se nalazi u argumentu konstruktora numberOfAgents. Ako je numberOfAgents izvan raspona onda se pretpostavlja da je 1. Statičkom metodom generateId() se generiraju jedinstveni identifikacijski brojevi agenata na način da se statička članska varijabla generatedId poveća za 1 nakon svakog stvaranja novoga agenta. Ako je broj agenata u višeagentskom sustavu veći od maksimalnog broja agenata, tada metoda addAgent() ništa ne radi. Članska varijabla agents se indeksira primjenom identifikacijskog broja agenta.

Apstraktna klasa Agent predstavlja određeni mobilni ili inteligentni agent. Članskim varijablama name i ID je definirano ime i identifikacijski broj agenta koji se ne može mijenjati nakon što je postavljen u argumentu konstruktora ID (primjenom statičke metode generateId() iz klase MultiAgentSystem). Apstraktnu metodu getAgentType() je potrebno nadjačati u izvedenim klasama na način da vraća string mobile agent (ako se radi o mobilnom agentu) ili string intelligent agent (ukoliko se radi o inteligentnom agentu). Pozivom metode toString() se vraća tekst s ispisom imena i identifikacijskog broja agenta. Primjer ispisa je ispod u primjeru korištenja.

Tip inteligentnog agenta je definiran enumeracijom IntelligentAgentType koja može imati dvije vrijednosti REASONER i LEARNER. Vrijednost članske varijable taskId u klasi MobileAgent predstavlja broj zadaće (koju mobilni agent odrađuje na udaljenom računalu) i postavlja se u konstruktoru primjenom metode doRemoteTask() koja generira slučajni cjelobrojni broj u intervalu [1, 10].

Metoda printSystemInfo() u klasi MultiAgentSystem ispisuje agente koji su članovi definiranog višeagentskog sustava. Pri tome se koriste pozivi metoda getAgentType() i toString() za ispis tipa, imena i identifikacijskog broja agenta. Ako je na nekom mjestu agent null taj ispis se preskače.

Primjer korištenja:

MultiAgentSystem mas = new MultiAgentSystem(4);

mas.addAgent(new MobileAgent("Speedy", MultiAgentSystem.generateId()));
mas.addAgent(new MobileAgent("Agile", MultiAgentSystem.generateId()));
mas.addAgent(new IntelligentAgent("Brainy", MultiAgentSystem.generateId(), IntelligentAgentType.REASONER));
mas.addAgent(new IntelligentAgent("Smart", MultiAgentSystem.generateId(), IntelligentAgentType.LEARNER));

mas.printSystemInfo();

System.out.println("---");
System.out.println(new MobileAgent("Fast", MultiAgentSystem.generateId()));
System.out.println(new IntelligentAgent("Clever", MultiAgentSystem.generateId(), IntelligentAgentType.REASONER));

Očekivani ispis:

mobile agent (name: Speedy, id: 0)
mobile agent (name: Agile, id: 1)
intelligent agent (name: Brainy, id: 2)
intelligent agent (name: Smart, id: 3)
---
(name: Fast, id: 4)
(name: Clever, id: 5)
