2. zadatak (7 bodova)

Vaš zadatak je napisati programski kod na način kako je to specificirano UML dijagramom klasa na slici.

Legenda vidljivosti u dijagramu klasa:

Napomena: Sve klase i sučelja imaju vidljivost postavljenu na package private i nalaze se u paketu hr.fer.oop.mtexam.task2.

Klasom MultiAgentSystem je oblikovan višeagentski sustav koji može sadržavati 0 ili više programskih agenata. Maksimalan broj agenata se nalazi u argumentu konstruktora numberOfAgents. Ako je numberOfAgents izvan raspona onda se pretpostavlja da je 1. Statičkom metodom generateId() se generiraju jedinstveni identifikacijski brojevi agenata na način da se statička članska varijabla generatedId poveća za 1 nakon svakog stvaranja novoga agenta. Ako je broj agenata u višeagentskom sustavu veći od maksimalnog broja agenata, tada metoda addAgent() ništa ne radi. Članska varijabla agents se indeksira primjenom identifikacijskog broja agenta.

Apstraktna klasa Agent predstavlja voditelja aukcije ili ponuditelje. Članskim varijablama name i ID je definirano ime i identifikacijski broj agenta koji se ne može mijenjati nakon što je postavljen u argumentu konstruktora ID (primjenom statičke metode generateId() iz klase MultiAgentSystem). Apstraktnu metodu getAgentType() je potrebno nadjačati u izvedenim klasama na način da vraća string auctioneer (ako se radi o voditelju aukcije) ili string bidder (ukoliko se radi o ponuditelju). Pozivom metode toString() se vraća tekst s ispisom imena i identifikacijskog broja agenta. Primjer ispisa je ispod u primjeru korištenja.

Tip strategije licitiranja ponuditelja je definiran enumeracijom BiddingStrategyType koja može imati dvije vrijednosti TRUTHFUL i SPECULATIVE. Vrijednost članske varijable startingPrice u klasi Auctioneer predstavlja početnu cijenu licitiranja i postavlja se u konstruktoru primjenom metode startAuction() koja generira slučajni cjelobrojni broj u intervalu [10, 20].

Metoda systemInfo() u klasi MultiAgentSystem ispisuje agente koji su članovi definiranog višeagentskog sustava. Pri tome se koriste pozivi metoda getAgentType() i toString() za ispis tipa, imena i identifikacijskog broja agenta. Ako je na nekom mjestu agent null taj ispis se preskače.

Primjer korištenja:

MultiAgentSystem mas = new MultiAgentSystem(4);

mas.addAgent(new Auctioneer("Loud", MultiAgentSystem.generateId()));
mas.addAgent(new Bidder("Wily", MultiAgentSystem.generateId(), BiddingStrategyType.TRUTHFUL));
mas.addAgent(new Bidder("Crafty", MultiAgentSystem.generateId(), BiddingStrategyType.SPECULATIVE));

mas.systemInfo();

System.out.println("---");
System.out.println(new Auctioneer("Boisterous", MultiAgentSystem.generateId()));
System.out.println(new Bidder("Wise", MultiAgentSystem.generateId(), BiddingStrategyType.TRUTHFUL));

Očekivani ispis:

auctioneer (name: Loud, id: 0)
bidder (name: Wily, id: 1)
bidder (name: Crafty, id: 2)
---
(name: Boisterous, id: 3)
(name: Wise, id: 4)
