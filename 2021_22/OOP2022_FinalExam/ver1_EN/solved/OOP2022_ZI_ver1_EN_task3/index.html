Assignment 3 (9 points)

You are tasked with helping the HR (Human Resources) department with employing new people. Your job is to rank people applied to certain positions based on the information from their CV. They are assigned a certain number of points (score) based on their skills and experience. Skills can be worth varying amounts of points depending on the position.

You need to finish the Position class by implementing the empty methods and by adding certain collections. Places where code is missing are marked with a // TODO comment. The class diagram is the following:

Position class

The Position class describes open position for which people apply. It contains the following attributes:

  • name - Name of the position.
  • requiredSkills - List of required skills and experiences for a position.
  • applications - Missing collection which has to be added. It contains Applications ordered descending based upon their number of points (score). If two applications have the same number of points they are then ordered by the ID of the applicant of the application.

Besides the applications attribute, you must also implement the following methods:

  • addApplication(Applicant) - Adds a new Application for a position by calculating the score of the Applicant argument, creating a new Application object and adding it to the applications collection.
  • summarizeApplications() - Returns a text which describes the order of the current applications for the position. The string consists of the name of the position and then a row for each of the applications. Each application row contains the name of the applicant and the score, rounded to two decimal places. However, if the score is 0 instead of it, we output a message: "did not meet requirements". Also, each row is indented by a single TAB (\t) character. For more information, look at the example.
  • getApplications() - Returns the collection of stored applications, ordered as described above. Used for testing.

Calculating the points (score attribute of Application)

Points/score of an Applicant for a Position are calculated by using the requiredSkills stored in the Position. For each skill we calculate the score by multiplying the applicant's experience/score in the skill (getRequiredSkillScore) with the skill's multiplier. The multiplier signifies how important a skill is for a position. Also, the applicant has to have at least the minimum or more score in each of the skills the Position requires, otherwise he gets a score of 0 in his Application.
