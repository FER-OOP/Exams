Opis zadatka

Zadatak je izgraditi objektni model za upravljanje računima korisnika u bankarskom sustavu. Ako nije drugačije specificirano, sve tipove je potrebno smjestiti u paket hr.fer.oop.

Sučelje Account reprezentira funkcionalnost korisničkog računa. Vidljivost ovog elementa omogućuje javni pristup. U sebi sadržava:

  • Metodu void performTransaction(int) koja izvršava transakciju i ažurira stanje računa. Transakcije mogu biti pozitivne (povećanje stanja računa) i negativne. Pristup ovog tipa je javan.

Razred User modelira korisnički račun. Nije moguće instancirati primjerke ove klase. Vidljivost ovog elementa omogućuje javni pristup. U sebi sadržava:

  • Varijablu name koja predstavlja ime i tipa je String. Pristup je moguć iz klasa istog paketa te iz klasa koje naslijeđuju taj element.
  • Varijablu balance koja modelira stanje računa i tipa je int. Pristup je moguć iz klasa istog paketa te iz klasa koje naslijeđuju taj element.
  • Varijablu password koja predstavlja lozinku i tipa je String. Element je vidljiv klasama iz istog paketa te klasama koje naslijeđuju taj element.
  • Konstruktor User(String, String, int) koji postavlja ime, lozinku i stanje računa. Pristup ovog tipa je javan.
  • Metodu String getName() koja je getter. Pristup ovog tipa je javan.
  • Metodu void setName(String) koja je setter. Vidljivost ovog elementa omogućuje javni pristup.
  • Metodu String getPassword() koja je getter. Element ima maksimalnu vidljivost.
  • Metodu int getBalance() koja je getter. Element ima maksimalnu vidljivost.
  • Metodu void setBalance(int) koja je setter. Element ima maksimalnu vidljivost.
  • Metodu void changePassword(String) koja omogućuje promjenu lozinke.
    • Element ima maksimalnu vidljivost.
    • Metoda nema tijelo.

Razred Individual reprezentira entitet individualnog (privatnog) korisnika. Element je vidljiv iz svih klasa. Podtip je od User. Implementira Account. Ima:

  • Varijablu limit koja predstavlja limit transakcije, tj. maksimalni iznos koji se može koristiti u transakciji (neovisno radi li se o uplati ili isplati) i tipa je int.
    • Potrebno je onemogućiti promjenu inicijalne vrijednosti i to na razini prevodioca.
    • Ovaj je element vidljiv samo iz vlastite klase.
  • Varijablu address koja specificira adresu i tipa je String. Ovaj je element vidljiv samo iz vlastite klase.
  • Konstruktor Individual(String, String, int, String, int) koji postavlja ime, lozinku, stanje računa, adresu i limit transakcije . Element je vidljiv iz svih klasa.
  • Metodu String getAddress() koja je getter. Vidljivost ovog elementa omogućuje javni pristup.
  • Metodu void setAddress(String) koja je setter. Vidljivost ovog elementa omogućuje javni pristup.
  • Metodu int getLimit() koja je getter. Pristup ovog tipa je javan.
  • Metodu void performTransaction(int) koja izvršava transakciju ako na računu ima dovoljno novaca te ako je apsolutni traženi iznos za transakciju manji/jednak limitu. Pristup ovog tipa je javan.
  • Metodu void changePassword(String) koja mijenja lozinku samo ako nova ima duljinu barem 6 znakova. Element ima maksimalnu vidljivost.

Primjer izvršavanja:


        
        Individual individual = new Individual("Pero Perich", "password123", 5000, "Address 1", 1000);
        
        individual.performTransaction(-2000);
        
        System.out.println(individual.getName() + " balance: " + individual.getBalance());
        
        individual.performTransaction(-1000);
        
        System.out.println(individual.getName() + " balance: " + individual.getBalance());
        
        individual.changePassword("newp");
        
        System.out.println(individual.getPassword());
        
        individual.changePassword("properpass234243234");
        
        System.out.println(individual.getPassword());
