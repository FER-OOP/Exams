Task Description

The task is to implement an object model for management of user accounts in the banking system. Unless stated otherwise, all the types are required to place in the package hr.fer.oop.

Interface Account specifies the user account functionality. Visibility of this element secures the public access. It is composed of:

  • Method void performTransaction(int) which executes the transaction and updates the balance. Transactions may be positive (increase of balance) and negative. The access of this type is public.

The class User specifies a user account. It is not possible to instantiate the objects of this class. Visibility of this element secures the public access. It is composed of:

  • Variable name which specifies the name and is of type String. The access is possible from the class of the same package and from the subclass.
  • Variable balance which models the balance and is of type int. The access is possible from the class of the same package and from the subclass.
  • Variable password which models the password and is of type String. The element is visible to clasess from the same paket as well as from the subclass.
  • Constructor User(String, String, int) which sets the name, password and the balance. The access of this type is public.
  • Method String getName() which is getter. The access of this type is public.
  • Method void setName(String) which is setter. Visibility of this element secures the public access.
  • Method String getPassword() which is getter. This element has the maximum visibility.
  • Method int getBalance() which is getter. This element has the maximum visibility.
  • Method void setBalance(int) which is setter. This element has the maximum visibility.
  • Method void changePassword(String) which enables the change of password.
    • This element has the maximum visibility.
    • The method does not have the body.

The class Individual models an individual (private) user entity. This element is visible from all the classes. It is a subtype of User. It implements Account. It has:

  • Variable limit which specifies the transaction limit, i.e. the maximum amount that can be used in a transaction (regardless of whether it is payment or payout) and is of type int.
    • It is important to prevent the modification of the initial value at the compiler level.
    • This element is visible only from its own class.
  • Variable address which represents the address and is of type String. This element is visible only from its own class.
  • Constructor Individual(String, String, int, String, int) which sets the name, password, balance, address and transaction limit. This element is visible from all the classes.
  • Method String getAddress() which is getter. Visibility of this element secures the public access.
  • Method void setAddress(String) which is setter. Visibility of this element secures the public access.
  • Method int getLimit() which is getter. The access of this type is public.
  • Method void performTransaction(int) which executes the transaction if there is enough balance and if the absolute amount in the transaction is less/equal to the limit. The access of this type is public.
  • Method void changePassword(String) which changes the password only if the new password has at least 6 characters. This element has the maximum visibility.

Sample code:


        
        Individual individual = new Individual("Pero Perich", "password123", 5000, "Address 1", 1000);
        
        individual.performTransaction(-2000);
        
        System.out.println(individual.getName() + " balance: " + individual.getBalance());
        
        individual.performTransaction(-1000);
        
        System.out.println(individual.getName() + " balance: " + individual.getBalance());
        
        individual.changePassword("newp");
        
        System.out.println(individual.getPassword());
        
        individual.changePassword("properpass234243234");
        
        System.out.println(individual.getPassword());
