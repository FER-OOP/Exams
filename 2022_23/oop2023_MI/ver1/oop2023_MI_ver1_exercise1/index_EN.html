Task Description

The task is to implement an object model for buildings. Unless stated otherwise, all the types are required to place in the package hr.fer.oop.

Interface EnergyEfficiency specifies the energy efficiency functionality. Visibility of this element secures the public access. It is composed of:

  • Method boolean investigateEnergyEfficiency() which returns true if the building is energy efficient, false otherwise. The access of this type is public.

The class Building specifies building. It is not possible to instantiate the objects of this class. It implements EnergyEfficiency. Visibility of this element secures the public access. It has:

  • Variable address which represents address and is of type String.
    • It is important to prevent the modification of the initial value at the compiler level.
    • The element is visible to clasess from the same paket as well as from the subclass.
  • Variable numberOfApartments which specifies the number of apartments and is of type int.
    • The access is possible from the class of the same package and from the subclass
    • It is important to prevent the modification of the initial value at the compiler level.
  • Variable energyConsumption which is the energy consumption and is of type int.
    • It is important to prevent the modification of the initial value at the compiler level.
    • The access is possible from the class of the same package and from the subclass
  • Constructor Building(String, int, int) which sets the address, number of aparments and energy consumption. This element is visible from all the classes.
  • Method String getAddress() which is getter. This element has the maximum visibility.
  • Method int getNumberOfApartments() which is getter. This element has the maximum visibility.
  • Method int getEnergyConsumption() which is getter. This element has the maximum visibility.
  • Method String calculateEnergyClass() which returns the energy class of a building based on energy consumption.
    • The method does not have the implementation.
    • This element has the maximum visibility.

The class ResidentialBuilding is a residential building. This element has the maximum visibility. It is a child of Building. It contains:

  • Variable numberOfParkingSpots which models the number of parking spots and is of type int. Visibility of this element secures the public access.
  • Constructor ResidentialBuilding(String, int, int, int) which sets the address, number of apartments, energy consumption and the number of parking spots. This element is visible from all the classes.
  • Method int getNumberOfParkingSpots() which is getter. Visibility of this element secures the public access.
  • Method boolean hasFreeParkingSpots() which returns true if the number of parking spots are greater than the number of apartments, false otherwise. Visibility of this element secures the public access.
  • Method String calculateEnergyClass() which calculates the energy class as follows: if the energy consumption if less than 50, the class is A, if less than 90 the class is B, else the class is C. This element is visible from all the classes.
  • Method boolean investigateEnergyEfficiency() which returns true if the energy class is A, else returns false. The access of this type is public.

Sample code:



       
        ResidentialBuilding rBuilding = new ResidentialBuilding("Address 1", 20, 500, 30);

        System.out.println("RESIDENTIAL BUILDING");
        System.out.println("Address: " + rBuilding.getAddress());
        System.out.println("Number of apartments: " + rBuilding.getNumberOfApartments());
        System.out.println("Number of parking spots: " + rBuilding.getNumberOfParkingSpots());
        System.out.println("Energy consumption: " + rBuilding.getEnergyConsumption());
        System.out.println("Energy class: " + rBuilding.calculateEnergyClass());
        System.out.println("Is energy efficient: " + rBuilding.investigateEnergyEfficiency());
        System.out.println("Has free parking spots: " + rBuilding.hasFreeParkingSpots());
