In the task, it is necessary to implement the program code of the classes Triple and SimpleTripleTreeSet.

The Triple class represents a collection of three Strings, in such a way that the first, second and third elements are defined in the class, but so that the instances of the Triple class are equal to each other regardless of the order of the elements. So a Triple with elements ("T","H","M") would be equal to ("H","M","T"). Also, Triple can contain null elements and duplicates.

In addition to the methods marked in the UML diagram, notice that the Triple class should implement the Comparable interface. To compare which of the two instances of the Triple class is larger, it is necessary to sort their elements according to natural order, and then compare the smallest element of the first instance with the smallest element of the second instance. In the case of equality, the middle elements of the two instances are compared, and again in the case of equality, the largest elements are compared. You must not change the private variables first, second and third.

SimpleTripleTreeSet represents a set of elements of type Triple. For successful implementation, SimpleTripleTreeSet should contain a sorted set of elements according to their natural order. A set by definition must not store duplicates. Your task is to implement the class according to the UML diagram without using the existing interface Set and the existing implementations of the interface Set (HashSet, TreeSet, ...). Instead you should use the existing implementation of the Map interface, which you will define as the private variable internalMap. Class SimplePairHashSet may only have one that one variable. The requested methods of the class SimpleTripleTreeSet should have identical functionality as the methods of the class TreeSet of the same name (https://docs.oracle.com/en/java/javase/17/docs/api/java.base/java/util/TreeSet .html).

Attention: All classes and interfaces have visibility set to package private and are in the hr.fer.oop.triple package. When implementing methods, you may use additional auxiliary data structures because the speed of program code execution is not important, i.e. the program code does not have to be optimized. Calling any method must not "destroy" the collection (ie delete all elements of the collection or similar).

Triple triple2 = new Triple("Marco", "Jacob", "Peter");

System.out.println("triple2 compared equal to triple3: " + (triple2.compareTo(triple3) == 0));
