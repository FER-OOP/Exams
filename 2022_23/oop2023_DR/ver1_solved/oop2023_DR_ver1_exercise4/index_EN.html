There is a Student class:

You need to write a Pair class that represents a pair of objects. The first element in pair must be a number(something that extends class Number, eg. Integer or Double), while second element could be of any type. Pair has fields first and second, which are private and have corresponding getters and setters. The constructor sets both attributes.

Furthermore, you need to create a class TableEntry which is a specific kind of pair, with the first element fixed to Integer, and second element still can be of any type.

Finally, class HTable contains a private field list - which is a list of TableEntry elements, where each second part of pair is a list of the same type on which HTable was parametrized. Eg. one can create HTable of Students, whare HTable contains a list of TableEntry elements, where each TableEntry contains an integer as the first element and second element is a list of students to which that integer is a hashCode. Eg. for students (1, Ana, Anic),(2, Tin, Maric),(3, Tomislav, Horvat),(4, Ivana, Grgic) and (5, Ana, Anic) "table" could look like this (bear in mind that method hashCode in Student class returns length of student name - ime.length):

3 , [(1, Ana, Anic) ; (2, Tin, Maric) ; (5, Ana, Anic)]
8 , [(3, Tomislav, Horvat)]
5 , [(4, Ivana, Grgic)]

(in this example you can see a list named list with three elements, each row representing one list element, object of the TableEntry type. First element in the TableEntry pair is hash (integer), and second element is a list of studenats which have that hash)

Class HTable must have a default constructor (without arguments), and should implement methods add and contains:

	public void add(T newElement);
	public boolean contains(T needle);

In case the element we are adding into the table is already in it, method add should throw exception, of DuplicateElementException type, with message "Element is already in HTable".

Finally, you need to create a DuplicateElementException class, which is unchecked, and has to have everything needed to show a given message.

Usage example:

		Pair<Double, String> p1 = new Pair<Double, String>(2.5, "aaa");
		Pair<Integer, Integer> p2 = new Pair<>(2, 2);
		//Pair<String, Integer> p3 = new Pair<>("bbb", 2); compile-time error!
		
		TableEntry<String> te1 = new TableEntry<String>(1, "ccc");
		//TableEntry<String> te2 = new TableEntry<String>(1.5, "ccc"); compile-time error!

		HTable<Integer> ht1 = new HTable<>();
		ht1.add(1);
		ht1.add(2);
		ht1.add(3);
		//ht1.add(1); // exception - Element is already in HTable
		System.out.println(ht1.contains(3)); //true
		System.out.println(ht1.contains(4)); //false
		
		
		Student s1 = new Student(1, "Ana", "Anic");
		Student s2 = new Student(2, "Tin", "Maric");
		Student s3 = new Student(3, "Tomislav", "Horvat");
		Student s4 = new Student(4, "Ivana", "Grgic");
		Student s5 = new Student(5, "Ana", "Anic");
		Student s6 = new Student(6, "Ana", "Anic");
		
		HTable<Student> ht2 = new HTable<>();
		
		ht2.add(s1);
		ht2.add(s2);
		ht2.add(s3);
		ht2.add(s4);
		ht2.add(s5);
		//ht2.add(s2); //exception - Element is already in HTable
		System.out.println(ht2.contains(s2)); //true
		System.out.println(ht2.contains(s4)); //true
		System.out.println(ht2.contains(s6)); //false (s6 != s1 i s6 != s5)
