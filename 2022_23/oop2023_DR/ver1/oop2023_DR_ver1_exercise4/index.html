Zadan je razred Student, kao na slici:

Potrebno je modelirati razred Pair koji predstavlja par objekata. Prvi element u paru mora nužno biti neki broj (tj. nešto što nasljeđuje razred Number, npr. Integer ili Double), dok drugi element može biti proizvoljnog tipa. Pair sadrži atribute first i second, koji su privatni te imaju odgovarajuće gettere i settere, a postoji i konstruktor koji postavlja oba atributa.

Nadalje, potrebno je modelirati razred TableEntry koji je specifična vrsta para, kojem je prvi element nužno Integer, a drugi proizvoljnog tipa.

Konačno, razred HTable sadrži privatni atribut list - tj. listu elemenata tipa TableEntry, gdje je svaki second dio para ustvari lista tipa po kojem je parametriziran razred HTable. Npr. moguće je stvoriti HTable studenata, tako što će HTable sadržavati listu elemenata tipa TableEntry, gdje svaki TableEntry sadrži cijeli broj i listu studenata kojima ja taj broj hashCode. Npr. za studente (1, Ana, Anic),(2, Tin, Maric),(3, Tomislav, Horvat),(4, Ivana, Grgic) i (5, Ana, Anic) "tablica" bi mogla izgledati ovako (uočite da je metoda hashCode u razredu Student prepravljena tako da kao rezultat vraća duljinu imena studenta):

3 , [(1, Ana, Anic) ; (2, Tin, Maric) ; (5, Ana, Anic)]
8 , [(3, Tomislav, Horvat)]
5 , [(4, Ivana, Grgic)]

(u primjeru je prikazana lista list s tri elementa, svaki redak predstavlja jedan element liste, tj. objekt tipa TableEntry. Prvi element para u TableEntry objektu je hash, a drugi element je lista studenata koji imaju taj hash)

Razred HTable treba imati podrazumijevani konstruktor (bez argumenata), te implementirati metode za dodavanje i pronalaženje elemenata u tablici:

	public void add(T newElement);
	public boolean contains(T needle);

U slučaju da u tablicu pokušavamo ubaciti element koji već postoji, metoda add treba baciti neprovjeravanu iznimku tipa DuplicateElementException, s porukom "Element is already in HTable".

Konačno, potrebno je napisati i razred DuplicateElementException koji spada u neprovjeravane iznimke, a mora sadržavati sve što je potrebno kako bi se mogla prikazati tražena poruka.

Primjer korištenja:

		Pair<Double, String> p1 = new Pair<Double, String>(2.5, "aaa");
		Pair<Integer, Integer> p2 = new Pair<>(2, 2);
		//Pair<String, Integer> p3 = new Pair<>("bbb", 2); compile-time error!
		
		TableEntry<String> te1 = new TableEntry<String>(1, "ccc");
		//TableEntry<String> te2 = new TableEntry<String>(1.5, "ccc"); compile-time error!

		HTable<Integer> ht1 = new HTable<>();
		ht1.add(1);
		ht1.add(2);
		ht1.add(3);
		//ht1.add(1); // exception - Element is already in HTable
		System.out.println(ht1.contains(3)); //true
		System.out.println(ht1.contains(4)); //false
		
		
		Student s1 = new Student(1, "Ana", "Anic");
		Student s2 = new Student(2, "Tin", "Maric");
		Student s3 = new Student(3, "Tomislav", "Horvat");
		Student s4 = new Student(4, "Ivana", "Grgic");
		Student s5 = new Student(5, "Ana", "Anic");
		Student s6 = new Student(6, "Ana", "Anic");
		
		HTable<Student> ht2 = new HTable<>();
		
		ht2.add(s1);
		ht2.add(s2);
		ht2.add(s3);
		ht2.add(s4);
		ht2.add(s5);
		//ht2.add(s2); //exception - Element is already in HTable
		System.out.println(ht2.contains(s2)); //true
		System.out.println(ht2.contains(s4)); //true
		System.out.println(ht2.contains(s6)); //false (s6 != s1 i s6 != s5)
