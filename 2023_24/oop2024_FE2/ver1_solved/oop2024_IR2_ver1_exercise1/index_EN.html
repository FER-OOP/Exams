In the directory tree there are files with different names and extensions. Your task is to write the FileRemoveSimulator class, enabling the following functionality:

  • the user can specify a list of extensions to be deleted
  • the user additionally specifies a list of names (full file name, together with the extension) of files that he DOES NOT want to delete

These two lists are given as arguments to the FileRemoveSimulator class constructor.

Additionally, the class must implement two methods:

List<String> getFilesMarkedForRemoval() //method that returns a list of names of all file to be deleted
List<String> getExcludedFiles() //method that returns a list of the names of all found files that have the extension from markedForRemoval list, the but are listed in the "skip" list.

eg suppose you have the following directory and file tree:

dir1:
 - file1.txt
 - file2.jpg
 - file3.jpg
 - file4.zip
 - dir2:
   - file5.txt

If the list of extensions contains ".txt" and ".jpg" and the list of filenames to skip is "file5.txt" and "file8.jpg" then the getFilesMarkedForRemoval() method should return the list:

	"file1.txt", "file2.jpg", "file3.jpg"

and method getExcludedFiles() returns:

	"file5.txt"

Note: The FileRemoveSimulator class must implement the FileVisitor interface and it must be placed in the hr.fer.oop.jir2024.zad1 package.

Note: To work with the files, be sure to use the java.nio.file.Path class.
