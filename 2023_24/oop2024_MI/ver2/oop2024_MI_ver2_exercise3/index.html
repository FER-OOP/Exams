Opis zadatka

Kratki opis domene: osobe s invaliditetom zbog raznih prepreka na koje nailaze često nemaju osigurane jednake mogućnosti. Kako FER pomaže osobama s invaliditetom i doprinosi postizanju uključivog društva možete vidjeti na stranicama ICT-AAC (opaska: proučiti stranicu ali tek nakon ispita! http://www.ict-aac.hr/).

Zadatak je napraviti objektni model za procesiranje teškoća koje imaju osobe s invaliditetom. Konkretno, u zadatku je potrebno implementirati dvije klase: DisabilityException i Solution. Format skupa podataka prikazan je u primjeru izvršavanja: svaka ispravna linija oblika je '[A] ### [B] #[C] --', gdje [A] označava ime osobe (pretpostavite da je uvijek ispravno), [B] označava teškoću (predefinirana vrijednost, pretpostaviti da teškoća može biti zapisana proizvoljnom kombinacijom malih i velikih slova, uputa: koristiti equalsIgnoreCase za provjere), [C] je razina oštećenja (decimalna vrijednost između 0.00% i 100.00% uključivo). Cilj je u klasi Solution (s praznim konstruktorom) implementirati sučelje DisabilityDatasetManager kako bi se parsirale sve linije teksta i obavile sve tražene funkcionalnosti. Potrebno je i napraviti provjeravanu iznimku DisabilityException. Pretpostaviti da će: skup podataka uvijek imati više linija, zaglavlje uvijek biti dobro napisano, imena osoba će uvijek postojati i biti ispravna (alfanumerički znakovi bez drugih znakova). Ako nije drugačije specificirano, sve tipove je potrebno smjestiti u paket hr.fer.oop.

Enumeracija Disability reprezentira teškoće koje mogu imati osobe s invaliditetom (vida, intelektualne, fizičke). Prepostaviti da se taksonomija teškoća neće mijenjati. Opaska: zbog sažetosti zadatka nisu navedene sve vrste teškoća..

Klasa DisabilityRecord reprezentira zapis teškoće s atributima personName (ime osobe s teškoćom), disability (teškoća), rating (razina oštećenja od 0 do 1 što predstavlja od 0% do 100%. Interval je [0.0, 1.0], odnosno, donja i gornja granica su uključive).

Razred DisabilityException je model provjeravane iznimke koja se baca kada dođe do problema kod procesiranja. Vidljivost ovog elementa omogućuje javni pristup. U sebi sadržava:

  • Konstruktor DisabilityException(String) koji postavlja poruku iznimke. Njen sadržaj je čitava linija teksta koja je neuspješno procesirana. Pristup ovog tipa je javan.

Sučelje DisabilityDatasetManager oblikuje skup metoda za procesiranje i analizu zapisa teškoća. Sadrži:

  • Metodu int processDataset(String) koja procesira ulazni skup podataka (uz pomoć processLine metode) koji je predan u tekstualnom formatu. Vraća broj neuspješno procesiranih zapisa teškoća. Vidljivost ovog elementa omogućuje javni pristup.
  • Metodu DisabilityRecord processLine(String) koja procesira liniju teksta iz skupa podataka te vraća instancu DisabilityRecord na temelju parsiranih podataka iz predane linije teksta. Pretpostaviti da teškoća može biti zapisana proizvoljnom kombinacijom velikih i malih slova (pomoć: koristiti equalsIgnoreCase). Obratiti pozornost na raspon razine oštećenja. Baca DisabilityException ako dođe do problema u procesiranju (parsiranje, vrijednost izvan raspona, linija nije prema propisanoj strukturi). Element je vidljiv iz svih klasa.
  • Metodu String maxRating(Disability) koja baca iznimku java.lang.UnsupportedOperationException bez poruke. Pristup ovog tipa je javan.
  • Metodu String share(Disability) koja na temelju završene obrade iz metode processDataset računa zastupljenost neke teškoće u odnosu na populaciju (npr. za VISUAL: [broj zapisa s vizualnim oštećenjima]/[ukupan broj zapisa]). Povratna vrijednost je u obliku Stringa koji je formatiran na jednu decimalu, npr.: 0.4321 će imati String sadržaja 43.2%. Ako nema podataka o navednoj teškoći ili se ne može izračunati vratiti 0.0%. Pomoć kako escapati znak posto kod formatiranja: %%. Element je vidljiv iz svih klasa.

Klasa Solution je rješenje zadatka koje je potrebno napraviti. Implementira DisabilityDatasetManager. Vidljivost ovog elementa omogućuje javni pristup.

Primjer izvršavanja:


		
		System.out.println("Please note the dataset is artificial, i.e., does not represent the real population.");

		String dataset = String.join("\n", 
				"Person_name ### Disability #% --",
				"Pero ### VISUAL #33.45 --", 
				"Marijeta ### Visual #10.22 --", 
				"Ana ### intellectual #15.78 --", 
				"Marko ### PHYSICAL #20.55 --",
				"Jasmina ### intellectual #5.67fgdgh56zzh --", // wrong
				"Sasha ### VISUAL #12.89 --", 
				"Luka ### HEARING #8.75", // wrong
				"Petra ### ", // wrong
				"Ivan ### PHYSICAL 18.92 --", // wrong
				"Marta ### VISUAL #28.14", // wrong
				"Elena ------> HEARING #14.67 --", // wrong
				"Nikola ### INTTTTTTTELEKETET #9.81 --", // wrong
				"Davor ### pHySiCAL #30.20 --", 
				"Katarina ### viSUAL #3.99 --");

		DisabilityDatasetManager manager = new Solution();
		int no = manager.processDataset(dataset);
		System.out.printf("Number of failed lines: %d \n", no);

		System.out.println("Share for physical disability: " + manager.share(Disability.PHYSICAL));
		System.out.println("Share for intellectual disability: " + manager.share(Disability.INTELLECTUAL));
		System.out.println("Share for visual disability: " + manager.share(Disability.VISUAL));

		// uncomment this to throw an exception:
		// manager.maxRating(Disability.INTELLECTUAL);
