In the task, it is necessary to convert a series of text lines in the form of an array of Strings into the list MyList, in which there are Product products.

MyList class is given, which is similar to the list we did on the recapitulation. It works with objects (class Object) which should be parameterized with generics. When doing so, you may receive a translation warning like this:

Compile errors/warnings:

The classes that need to be created are shown in the UML class diagram. In all classes, it is necessary to implement attributes (member variables), constructors and methods as shown in the diagram. You are free to add and implement additional methods, attributes and constructors.

Visibility legend in class diagram:

The Price class represents the price of a product and consists of currency which represents the currency and amountInHundredths which represents the value. The value is an integer that represents hundreds of parts of a currency. For example, 4.35 EUR is written as the value 435 in amountInHundredths and EUR in currency. Objects of this class are immutable, i.e. there are no set methods and after the object is created, its member variables cannot be changed.

The Product class represents a product. A product consists of a name, a category and a price. Objects of this class are immutable, i.e. there are no set methods and after the object is created, its member variables cannot be changed.

It is necessary to create a hierarchy of exceptions as given in the diagram. The ProductParseException class is unchecked exception. It represents all exceptions that can occur when converting rows into products (parsing procedure). The ProductParseCollectorException class collects all exceptions in a list. The addReason method adds a reason, and the getReasons method returns a list of reasons. The ProductParseRowException class represents an exception that represents an error when processing a single row, and the ProductParseCellException class represents a single cell exception. What the getMessage method should return in the ProductParseException, ProductParseRowException and ProductParseCellException classes is visible in example code.

The ProductTableParser class is used for parsing rows and has two methods:

  • parseTable - starts parsing and if everything is fine it ends normally. If there is a problem in parsing, it collects all exceptions and throws ProductParseCollectorException in which all exceptions are collected. The order of the columns is always the same: product name, category and price. If there are more columns than necessary the others are ignored. The price format is a number with a decimal point, one space and currency.
  • getProducts - returns a list of products that were successfully parsed by the last call to the parseTable method, i.e. parsed lines without exceptions

Note: all classes are in the hr.fer.oop package and are public.

Usage example:

public static void main(String[] args) {
  System.out.println(new ProductParseException("some error").getMessage());

  System.out.println(new ProductParseRowException(7, "wrong number of columns").getMessage());
  System.out.println(new ProductParseRowException(8).getMessage());

  System.out.println(new ProductParseCellException(9, 3, "can not parse price").getMessage());
  System.out.println(new ProductParseCellException(10, 4).getMessage());

  System.out.println("---");

  String[] lines = {
      "+---------+----------+----------+",
      ": coffee  : drink    : 2.4 EUR  :",
      "+---------+----------+----------+",
      ": phone   : electronics",
      "+---------+----------+----------+",
      ": car    : vehicle : 30000 :",
      "+---------+----------+----------+",
      ": bike   : vehicle : 2000EUR :",
      "+----------+----------+----------+",
      ": computer : electronics : 1K EUR :",
      ": milk    : drink    : 2 USD  : some comment",
      "+---------+----------+----------+"
  };

  ProductTableParser parser = new ProductTableParser();
  try {
    parser.parseTable(lines);
  } catch (ProductParseCollectorException e) {
    System.out.println("Parse exceptions:");
    MyList<ProductParseException> reasons = e.getReasons();
    for (int i = 0; i < reasons.size(); i++) {
      ProductParseException reason = reasons.elementAt(i);
      System.out.println(reason.getMessage());
    }
  }

  System.out.println("---\nParsed products");
  MyList<Product> products = parser.getProducts();
  for (int i = 0; i < products.size(); i++) {
    Product p = products.elementAt(i);
    printProduct(p);
  }

}

private static void printProduct(Product p) {
  System.out.print("Product[name: " + p.getName());
  System.out.print(", category: " + p.getCategory());
  System.out.println(", price: " + (p.getPrice().getAmountInHundredths()/100.0) + " " + p.getPrice().getCurrency() + "]");
}
